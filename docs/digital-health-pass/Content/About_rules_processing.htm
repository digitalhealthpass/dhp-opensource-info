About rules processing

The rules engine evaluates credential and rule settings to determine whether a credential is valid when scanned. Processing is hierarchic using a "true, false" logic, assessing specification settings, then rule settings. The following are basic examples of results from verification scans, based on the applied rules processing of verifier configurations.

Example 1: Valid (green) scan result

Credential holder: Presents a SMART Health Card, has completed an approved test, has a negative test result

Rules: Enabled values for SMART Health card, check for approved test, check for negative test result

Scan result: Valid credential

Example 2: Invalid (red) scan result

Credential holder: Presents a SMART Health Card, has completed an approved test, has a positive test result

Rules: Enabled values for SMART Health card, check for approved test, check for negative test result

Scan result: Invalid credential

Example 3: Invalid (red) scan result

Credential holder: Presents a SMART Health Card, has completed an approved test, has a negative test result

Rules: Disabled value for SMART Health card

Scan result: Invalid credential
