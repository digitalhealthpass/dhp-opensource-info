Changing a verifier configuration

You can configure settings in a verifier configuration, for example, to set the configuration version.

About caching

With verifier configurations, caching occurs during signature verification. Enabling improves performance. Needed data includes the public keys used to validate credential signatures. Data is cached and refreshed as appropriate from the Verifier Configuration API.

  • Mobile apps always cache API calls, and refresh based at the specified Refresh rate.

  • For all supported credential types, NodeJS pre-caches authorization tokens, public keys for signature validation, business rules for credential validation, display mappings, and trust lists. NodeJS caches in memory on the computer that runs the application that uses the NodeJS SDK.

To change a verifier configuration:

  1. Log in to the administrative console.
  2. On the home page, click the Set up configurations tile.
  3. On the Configuration catalog page, click the <Company> tab.
  4. On the configuration page, under Credential Type, click the type that you want.
  5. Click Update to my catalog.
  6. In the Configuration dialog box, change settings as needed. For information about settings, see Adding a verifier credential.
  7. Click Save configuration.
