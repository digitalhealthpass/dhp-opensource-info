About Basic customers and Enterprise customers

The solution is flexible, and scales to meet customer size and needs.

Planning and administration

As a customer, plan both your deployment and your approach to administration based on your overall organization size and type. Consider the number of locations you have and the number of verifiers that you'll use.

Example

For , an example of a Basic customer might be a restaurant. Because there's a single physical location, with one entry and few staff set up as verifiers, the Customer Administrator can manage verifiers using the default organization, which represents a single physical location.

An example of an Enterprise customer might be a concert promoter. Let's say that this promoter manages several venues. Here, the deployment is better suited to setting up an organization at each venue. Each of these organizations can in turn manage verifier credentials, for example, to set up a verifier credential for each gate at the venue.

Note: This table lists general criteria for you to consider when getting ready for administration. Because customer needs vary greatly, use these as high-level guidelines.
