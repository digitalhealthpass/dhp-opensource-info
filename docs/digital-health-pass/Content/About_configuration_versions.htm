Master configurations

The Configuration Catalog includes these master configurations:

Complete ruleset

This configuration includes all rules, which are enabled with the exception of the expirationDate check.

Return-to-Work Configuration (Data Exchange Service)

This configuration is a copy of Complete Ruleset, but with display configurations updated for Data Exchange Services.

Signature and Expiration check only

This configuration is a copy of Complete Ruleset, with most rules disabled. The only enabled rules check the expirationDate field. Since the specification includes all rules, this configuration also includes all value sets. This means that if you enable any of those rules, the rules will work normally with parameters from those value sets.

Releases of administration

Version 1

In the Version 1 release, administrators had a read-only view of verifier configurations. While System Administrators and Customer Administrators, could add configurations to the Configuration Catalog, the configurations were used as-is.

Version 2 and subsequent releases

enhanced verifier configurations to support editing rule sets. For example, administrators can enable support for credential specifications (for example, Digital Health Pass or SMART Health Card), and the values for specific rules. Note that configurations are listed chronologically, with the most-recently updated configuration first.

Updates and versions

With each update, a verifier configuration is assigned an incremented version number. You can use configuration versions to track updates in the Configuration Catalog. The catalog displays the latest version of any configuration, whether it's a master or customer-specific configuration.

Updating version numbering with verifier configurations
Update Version numbering Example
Modifies rule values Changes last digit in version number

Administrator edits a version 1.0.0 configuration, then updates the rules and saves the configuration.

The administration console updates the configuration to version 1.0.1.

Disables a rule Changes second digit in version number

Administrator edits a version 1.1.1 configuration, then disables a rule and saves the configuration.

The administration console updates the configuration to version 1.2.1.
