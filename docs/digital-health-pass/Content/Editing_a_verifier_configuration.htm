Editing a verifier configuration

Complete limited updates to verifier configurations.

Note: Before proceeding, see Working with the rules editor to learn about rule sets in configurations.

To edit a verifier information:

  1. Log in to the administrative console.
  2. On the home page, click the Manage customers tile.
  3. On the Customers page, click the customer that you want.
  4. On the Organizations page, under Actions, click the open icon (arrow).
  5. On the Verifier Credentials page, click Configuration Catalog.
  6. In the catalog, locate the configuration that you want.
  7. For the configuration, under Actions, click open model icon (arrow).
  8. To enable or disable a credential:
    1. Click the Config Rules tab.
    2. Expand the category that you want (click the triangle icon).
    3. As needed, select a check box to enable, and clear a check box to disable.
  9. To update a value:
    1. Click the Config Values tab.
    2. In the navigation, click the appropriate item, for example, Accepted Pass Status.
    3. Review the item details. Make sure that you have the correct item and the value that you want.
    4. Next to Values, click the pencil icon.
    5. In the edit dialog, update the values or descriptions as needed.

      To delete a value, click the trash icon next to it.
    6. To add a value, click Add Value.
    7. Click Save.
  10. To update a rule:
    1. Click the Config Rules tab.
    2. Expand the category that you want (click the triangle icon), then click the specification that you want. Drill down to locate the rule.
    3. As needed, select a check box to enable, and clear a check box to disable.
  1. Click Save changes. In the dialog box that appears:
    1. Type a configuration name.

      Note that you cannot use an existing configuration name when you save an updated version.
    2. Select the appropriate Save... or Modify... option, for example, Save as new customer configuration.
    3. To inherit any changes from the configuration was copied, select the Receive updates to configuration check box. To not inherit such changes, clear the check box.

The administration console saves the configuration to the customer's catalog.

Note: You cannot roll back and use an earlier version of a verifier configuration.
